Step 2 — Install R packages

Paste this into the RStudio Console:

install.packages(c(
  "pavo",       # color + pattern stats
  "ape",        # tree handling
  "phytools",   # phyl.pca, Blomberg's K, Pagel's lambda
  "caper",      # PGLS
  "nlme",       # gls fallback for PGLS
  "plotrix",    # phytools dependency (don't skip)
  "png",        # PCA biplots embed exemplar PNG thumbnails as point markers
  "tidyr",      # pivot_longer in Part 3 plots
  "ggplot2",
  "remotes"     # used to install rfishbase 3.1.10 below
))

# rfishbase 3.1.10 — pinned because the current CRAN release pulls in
# duckdb, which needs a C++ toolchain that fails to build on many laptops.
# 3.1.10 exposes the same species() and ecology() calls we use in §2.8.
remotes::install_version("rfishbase", version = "3.1.10",
                         repos = "https://cloud.r-project.org",
                         upgrade = "never")

Then verify everything loaded cleanly:

library(pavo); library(ape); library(phytools); library(caper)
library(nlme); library(plotrix); library(tidyr); library(ggplot2)
stopifnot(requireNamespace("png", quietly = TRUE))         # biplot thumbnails
stopifnot(packageVersion("rfishbase") == "3.1.10")          # §2.8 FishBase pull
cat("YOU'RE READY FOR WEDNESDAY!\n")

If anything fails — especially pavo or caper, which have C dependencies — email your TA tonight, not Wednesday morning.

Step 4 — Readiness check

Run this short script in the Console — it confirms the tree loads, the species list matches the tree tips, and pavo can read one of the bundle images:

library(ape); library(pavo)

bundle <- "chaetodontidae-mini-bundle"

# The worksheet defines two small helpers in scripts/measurement-error.R
# that handle NEXUS vs Newick trees and strip specimen suffixes off tip
# labels. We don't need them for the readiness check — just the raw
# bundle contents.
tree_path      <- list.files(bundle, pattern = "-mini-tree\\.tre$", full.names = TRUE)
tree           <- ape::read.tree(tree_path)
picked_species <- readLines(file.path(bundle, "picked_species.txt"))

# Sanity: every directory in images/ matches a picked species
species_dirs <- list.dirs(file.path(bundle, "images"), recursive = FALSE)
stopifnot(setequal(basename(species_dirs), picked_species))

# pavo can read at least one image
first_sp_dir <- species_dirs[1]
first_png    <- list.files(first_sp_dir, pattern = "\\.png$", full.names = TRUE)[1]
img1 <- getimg(first_png)
stopifnot(!is.null(img1))

cat("READINESS CHECK PASSED — bundle has",
    length(picked_species), "species,",
    Ntip(tree), "tips on tree, pavo reads images.\n")

You should see “READINESS CHECK PASSED”. If you see anything else, re-download the bundle or email your TA.

Step 6 — One-click server smoke check

Wednesday’s analysis depends on FishView’s Export Bundle button actually shipping you a ZIP. We’ve got a special pre-class smoke-test endpoint that bypasses the in-class validation (the ≥15-species / ≥5-genera / error-species rules) so you can verify the whole pipeline reaches you tonight without spoiling the in-class curation work.

Three minutes, no analysis involved:

  1. Logged into your family’s dashboard from Step 5 — open any species in the table.

  2. On its review page, click Include on one or two images. (One is enough. You don’t need to star an exemplar, mark an error-species, or do any real curation — that’s all for Wednesday.)

  3. In a fresh browser tab, visit this URL with your family code in the path:

    http://164.67.78.200:5052/api/export-bundle/<your-family>?smoke=true

    Substitute <your-family> from this table:

    Group URL family code
    1 (triggerfishes) balistoidea
    2 (angelfishes) pomacanthidae
    3 (parrotfishes) scarinae
    4 (fairy wrasses) cirrhilabrini
    5 (surgeonfishes) acanthuridae
    6 (hogfishes) hypsigenyinae
    7 (Coris-allies) halichoerine

    So group 5 would visit http://164.67.78.200:5052/api/export-bundle/acanthuridae?smoke=true.

  4. You should get a download prompt for <family>-mini-bundle.zip (a few hundred KB to a few MB).

  5. Delete the ZIP after the download finishes — it’s a smoke artifact, not what you’ll analyze on Wednesday.

Expected outcomes

  • You get a ZIP → :white_check_mark: server pipeline works end-to-end; you’re ready for Wednesday.
  • Browser shows {"error":"nothing_to_export", ...} → you didn’t click Include on any image. Go back to step 2.
  • Browser shows a login page → your session expired. Re-enter your group code from Step 5 and retry.
  • Page won’t load at all → you’re off UCLA WiFi / VPN. Reconnect and retry.
  • Any other JSON error → email your TA with the error text by Tuesday afternoon.

If something breaks

  • Package install fails on pavo or caper: likely a system-library issue. On macOS try brew install gdal first, then re-install. On Windows try the binary build via install.packages("pavo", type = "binary").
  • read.tree() errors with “incorrect string constant”: the tree file didn’t fully download. Re-run Step 3.
  • getimg() errors with “PNG read failed”: an image in the bundle is malformed. List the bad path with list.files("chaetodontidae-mini-bundle/images") and email your TA.
  • Mini-bundle code returns “unknown group”: the FishView entry might not be fully provisioned yet. Email your TA — your group’s mini-bundle gets locked in by 6 PM Tuesday.
